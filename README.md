# GraphSolution

## Условие задачи
Задана система двусторонних дорог. Определить, можно ли, построив еще три новые дороги, добраться из заданного города до каждого из остальных городов, проезжая расстояние не более Т единиц.
## Исходные данные: 
1. Путь до файла входных данных в параметрах запуска (необязательно)

2. Количество городов

3. Количество односторонних дорог

4. Сами дороги (откуда — куда)

5. Рассматриваемый город

6. Максимально допустимая длина пути
## Выходные данные:
1. Память, затрачиваемая на хранение графовой структуры

2. Память, затрачиваемая на хранение возможных комбинаций рёбер

3. Ответ на вопрос из задачи

4. Граф (с добавленными рёбрами, если ответ положительный)

5. Время вычисления

В случае ошибки выводится соответствующее сообщение.
## Задача, реализуемая программой
Определение, можно ли достроить три односторонние дороги для того, чтобы расстояние от заданного города до остальных составляло не более заданного.
## Способ обращения к программе
Через терминал (app.exe).
## Возможные аварийные ситуации и ошибки пользователя
Неверное количество аргументов при запуске программы. Ошибка открытия тектового файла, ошибка чтения/записи в файл или из терминала. Нехватка памяти для инициализации массива.
## Матричное представление графа
Было выбрано, исходя из удобства выделения/освобождения памяти, а также для быстрого доступа к вершинам графа.
## Алгоритм Дейкстры
Был выбран, поскольку, согласно задаче, необходимо найти кратчайшие пути из одной вершины во все остальные, а рёбра не принимают отрицательных значений.
## Описание алгоритма
В начале работы программы происходит считывание графа по рёбрам, на основе пользовательского ввода строится матрица смежности. Затем в построенной матрице подсчитывается количество отсутствующих рёбер; на основе этого значения, методами комбинаторики вычисляется количество возможных комбинаций добавлений рёбер по 3 штуки. Далее путём обхода матрицы смежности заполняется данный массив.
После этого для каждой комбинации построений рёбер на графе применяется алгоритм Дейкстры, для получения расстояний из начальной вершины до остальных. Результаты записываются в массив, если все элементы массива меньше или равны максимально-допустимому расстоянию, эта комбинация дорог считается решением. Если в ходе перебора комбинаций подходящей не найдено, значит таковой не существует.
